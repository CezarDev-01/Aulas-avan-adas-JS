Laços de Repetição em JavaScript

Laços de repetição são frequentemente usados ao trabalharmos com arrays e são úteis para executar blocos de código repetidamente.

  • for: Usado quando o número de iterações é conhecido. Ele inclui inicialização, condição e incremento/decremento. 
    for (inicialização; condição; incremento) {
  // bloco de código
}

    Exemplo 1: Imprimir números de 1 a 5.

    for (let i = 1; i <= 5; i++) {
  console.log(i);
}
  • while: Executa o bloco de código enquanto a condição for verdadeira. Útil quando o número de iterações é desconhecido. 
    while (condição) {
  // bloco de código
}

    Exemplo 2: Contagem regressiva de 5 a 1.

    let i = 5;
while (i > 0) {
  console.log(i);
  i--;
}
  • do...while: Executa o bloco de código pelo menos uma vez, pois a condição é verificada após a primeira execução. 
    do {
  // bloco de código
} while (condição);

    Exemplo 1: Imprimir número aleatório até que seja maior que 0.7.

    let numero;
do {
  numero = Math.random();
  console.log(numero);
} while (numero <= 0.7);

    Exemplo 2: Solicitar senha até que seja correta.

    let senha;
do {
  senha = prompt("Digite a senha:");
} while (senha !== "1234");
alert("Senha correta!");
  • for...of: Usado para iterar sobre valores de objetos iteráveis, como arrays e strings. Ele acessa diretamente os valores. 
    for (const valor of objetoIterável) {
  // bloco de código
}

    Exemplo 1: Imprimir cada caractere de uma string.

    const palavra = "JavaScript";
for (const letra of palavra) {
  console.log(letra);
}

    Exemplo 2: Somar valores de um array.

    const numeros = [1, 2, 3, 4, 5];
let soma = 0;
for (const numero of numeros) {
  soma += numero;
}
console.log("Soma total:", soma);
  • for...in: Itera sobre as propriedades enumeráveis de um objeto. Ideal para objetos, mas não recomendado para arrays. 
    for (const propriedade in objeto) {
  // bloco de código
}

    Exemplo 1: Imprimir propriedades e valores de um objeto.

    const pessoa = { nome: "Ana", idade: 30, cidade: "São Paulo" };
for (const propriedade in pessoa) {
  console.log(propriedade, ":", pessoa[propriedade]);
}

    Exemplo 2: Contar caracteres em uma string.

    const frase = "Olá, mundo!";
const contagem = {};
for (const letra of frase) {
  contagem[letra] = (contagem[letra] || 0) + 1;
} 
console.log("Contagem de caracteres:", contagem);

Comandos Especiais nos Laços

  • break: Interrompe o laço e sai imediatamente. Útil quando queremos parar a execução ao atingir uma certa condição. 
    for (let i = 1; i <= 10; i++) {
  if (i === 5) {
    break; // Para o loop quando i é igual a 5
  }
  console.log(i);
}
  • continue: Interrompe a iteração atual e passa para a próxima, ignorando o restante do código dentro do laço para aquela iteração. 
    for (let i = 1; i <= 10; i++) {
  if (i % 2 === 0) {
    continue; // Pula números pares
  }
  console.log(i); // Imprime apenas números ímpares
}

Evitar Loops Infinitos

É essencial garantir que os laços tenham uma condição de término válida para evitar loops infinitos que podem travar a execução do código.

let i = 0;
while (i < 5) {
  console.log(i);
  i++; // Certifique-se de incluir um incremento ou uma condição de parada
}

Diferença entre for...of e for...in

for...of: Usado para iterar sobre valores de objetos iteráveis, como arrays e strings.

for...in: Usado para iterar sobre propriedades de objetos, mas não é recomendado para arrays, pois pode gerar comportamentos inesperados.

// Exemplo com for...of
const array = [10, 20, 30];
for (const valor of array) {
  console.log(valor); // Imprime os valores: 10, 20, 30
}

// Exemplo com for...in
const pessoa = { nome: "Carlos", idade: 25, cidade: "Brasília" };
for (const propriedade in pessoa) {
  console.log(propriedade, ":", pessoa[propriedade]);
}

forEach com Arrays

O método forEach é usado para iterar sobre arrays. Ele executa uma função para cada elemento do array e é útil em JavaScript moderno.

const numeros = [1, 2, 3, 4, 5];
numeros.forEach(numero => {
  console.log(numero); // Imprime cada número
});

Performance dos Laços

Para grandes volumes de dados, o laço for tradicional pode ser mais rápido em comparação a forEach e for...of, embora a diferença seja pequena para casos menores.
